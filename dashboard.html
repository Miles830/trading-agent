Submit a Trade Suggestion

Share a ticker, theme, or trade idea. Submitting opens a prefilled GitHub Issue tagged user-suggestion on the Miles830/trading-agent repo. Each Pre-Market routine reads open suggestion issues, scores them through the 6-agent framework, and either acts (within guardrails) or replies with reasoning, then closes the issue. Time-sensitive suggestions also get picked up by the Mid-Morning routine.

⚠ Suggestions become public on the GitHub repo. Don't include anything you wouldn't share publicly. You'll need to be signed in to GitHub to post — the form opens a prefilled issue page.

How it works
  1. Click Open prefilled GitHub issue — a new tab opens with the issue body filled in.
  2. If not signed in, GitHub will prompt you. Then click Submit new issue to post.
  3. The next Pre-Market routine (8:00 AM ET, weekdays) reads all open user-suggestion issues, scores each through the 6-agent framework, and decides:
    • APPROVED — passes Master gate; MOO order placed within guardrails.
    • DEFERRED — partial pass or guardrail conflict; reasoning posted.
    • REJECTED — fails gate decisively; blocking agents named.
  4. The agent comments on the issue with the decision and closes it. You'll get a GitHub email.
  5. If urgency is "Today", the Mid-Morning routine (11:00 AM ET) does a same-day check.
  6. Suggestions count toward the 3 MOO/day cap and all guardrails. They don't get a special pass — your thesis is one input among the 6 agents.
